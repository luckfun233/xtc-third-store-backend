# 变更文档：部署与请求规则（2026-03-07）

本文用于说明当前仓库的部署方式、请求规则、上架流程与兼容约束。

## 1. 本次主要变更

1. 新增 GitHub Pages 自动部署工作流：
   - `.github/workflows/deploy-pages.yml`
2. 部署触发规则改为：
   - `main` 或 `master` 分支任意 push 自动部署
3. 包目录结构简化为：
   - `packages/<category>/*.rpk`
4. `build_index.py` 自动推断 `rpkPath` 只按分类目录检索（不再自动推断旧的 appId/版本层级）。
5. `STORE_BASE_URL` 默认值设置为：
   - `https://store.1357924680liu.dpdns.org`
   - 若仓库变量 `STORE_BASE_URL` 已配置，则优先使用变量值。

## 2. 当前部署方式

### 2.1 工作流

- 索引重建：`.github/workflows/rebuild-index.yml`
  - 触发：`main/master` 分支中 `apps/**/*.json`、`packages/**/*.rpk`、`scripts/build_index.py` 变更
  - 作用：重建并提交 `data/index.json`

- Pages 发布：`.github/workflows/deploy-pages.yml`
  - 触发：`main/master` 分支任意 push
  - 作用：
    1) 构建 `data/index.json`
    2) 打包并发布以下目录到 GitHub Pages：
       - `apps/`
       - `assets/`
       - `data/`
       - `packages/`

### 2.2 域名优先级

1. 仓库变量 `STORE_BASE_URL`（最高优先）
2. 默认值 `https://store.1357924680liu.dpdns.org`

> 建议在仓库 Variables 中明确配置 `STORE_BASE_URL`，便于后续迁移域名。

## 3. 请求规则（前端访问方式）

## 3.1 应用索引

- 地址：`<STORE_BASE_URL>/data/index.json`
- 示例：`https://store.1357924680liu.dpdns.org/data/index.json`

## 3.2 包下载

- 地址：`<STORE_BASE_URL>/packages/<category>/<filename>.rpk`
- 示例：`https://store.1357924680liu.dpdns.org/packages/games/wearnote.rpk`

## 3.3 资源文件

- 图标：`<STORE_BASE_URL>/assets/...`
- 元数据：`<STORE_BASE_URL>/apps/...`

## 3.4 JSON 结构

`data/index.json` 结构保持不变（兼容旧前端）：
- 保留 `download.primary`
- 保留 `download.mirror`
- 保留 `download.proxy`
- 保留 `meta.primary`、`meta.mirror`

当前策略下 `primary/mirror/proxy` 默认可指向同一域名路径。

## 4. 上架规则（简化后）

1. 上传包到：`packages/<category>/`
2. 建议命名：`<appId>-<versionName>.rpk`
   - 例如：`sample-math-1.0.0.rpk`
3. 新建或更新：`apps/<category>/<appId>.json`
4. push 到 `main/master` 后自动重建索引并自动部署

## 5. rpkPath 自动推断规则

当应用元数据未填写 `rpkPath` 时：

1. 在 `packages/<category>/` 下优先精确匹配：
   - `<appId>.rpk`
   - `<appId>-<versionName>.rpk`
   - `<appId>_<versionName>.rpk`
2. 若精确匹配失败，尝试模糊匹配（文件名同时包含 `appId` 与 `versionName`）。
3. 若仍失败，尝试只按 `appId` 匹配。
4. 匹配到多个候选会报错，要求显式填写 `rpkPath`。

## 6. 对接建议

- 前端每次进入商店先拉取 `/data/index.json`。
- 下载失败可重试同地址（无需切镜像）。
- 如担心 CDN/浏览器缓存，可在索引请求后追加时间戳参数（例如 `?t=unix`）。

## 7. 运维检查清单

1. GitHub Pages Source 选择 GitHub Actions。
2. Actions 权限允许写入 Pages。
3. 仓库变量已设置：`STORE_BASE_URL`。
4. 自定义域名 DNS 已正确解析到 GitHub Pages。
